This endpoint is designed for templates without complex headers but supports parameters for customization. Simply plug in and configure your parameters for easy and efficient use.

#### Request Body Parameters

<ParamField path="from" type="string" required>
  The number registered with WhatsApp business from which the message is to be
  sent. Ensure that the country code is prefixed to the number. For example, if
  the number is in the India, the format would be: +91XXXXXXXXXX
</ParamField>

<ParamField path="templateName" type="string" required>
  This template name contains only lowercase alphanumeric characters and
  underscores.
</ParamField>

<ParamField path="to" type="string" required>
  Recipient's Mission
</ParamField>

<ParamField path="type" type="string" required>
  WhatsApp message templates type
</ParamField>

<ParamField path="components" type="object" required>
  To build a message content object for WhatsApp, typically include various
  fields such as param, header type(text, images, video, and other media).
</ParamField>

<ParamField path="body" type="object" required>
  Template body
</ParamField>

<ParamField path="params" type="string" required>
  If you have a registered template with placeholders and you need to submit
  parameter values in the same order as in the template, you can represent this
  as an array of values. Ensure that the values are not null, and if the
  template was registered without placeholders, you can submit an empty array.
</ParamField>

<CodeGroup>

```json SampleRequest.json
curl -G - /v1/whatsapp\
-H 'apikey':'string'
-H "Content-Type: application/json" \
-d '{
    "from": "91XXXXXXXXXX ",
    "campaignName": "api-test",
    "to": "91XXXXXXXXXX",
    "templateName": "template_name",
    "components": {
        "body":{
        "params":["var1", "var2", "var3"]
        }
    },
    "type": "template"
}'

```
